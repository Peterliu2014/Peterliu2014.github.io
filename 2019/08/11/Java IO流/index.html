Java IO流

发表于 2019-08-11 17:37:04 | 更新于: 2019-08-11 | 分类于 Java | 阅读数
字数统计: 2.5k | 阅读时长 ≈ 10

流是【传输数据】的一种方式，传输的数据基本单位是字节，为了处理的方便，在字节的基础上引入了字符的概念（一个字符包含多个字节），所以按照处理单位的不同，有字节流和字符流的说法。从流的方向来分类的话，又有输入流和输出流之分。

  • 字节流和字符流：一次处理一个字节的流为字节流，一次处理一个字符的流为字符流。
  • 输入流和输出流：如果程序是要写数据，那么就要调用相应的输出流。反之，如果要去读数据，调用输入流。
  • 数据源和目标媒介：输入流对应数据源，即数据从哪里读取。输出流对应目标媒介，即数据写入到哪里。文件、网络、内存缓存、管道都可以是数据源/目标媒介。

    管道流

    管道流用于两个线程间通信，在两个线程间以流的方式传递数据。

    字节流

    PipedOutStream

    PipedOutputStream3个write方法(继承自OutputStream)：
    * 1）write(int b)   将b的低八位写入流（高24位直接丢弃），就是一个字节。
* 2）write(byte[] b) -> write(b,0,b.length)  某个字符的编码值只有查询相应的编码表才能得到，有些麻烦。
* 因此提供了这个字节数组的方法。直接利用"好".getBytes()获得对应的字节数组之后，
* 调用write(byte[])将字符写入流中。
* 3）write(byte[] b,int offset,int len):将字符数组中从offset位置开始的len个字节数据写入流。
* 4）后两个方法最终还是调用write(int b),完成流数据的写入。

    PipedInputStream

    在utf-8的编码中汉字一般占用3个字节，特殊的汉字占用4个字节。
    PipedInputStream的read方法（继承自InputStream）
1）read()：一次读取一个【字节】，返回值是读取的【字节数据】。当返回-1时表示流中没有数据。
2）read(byte[] b)->read(b,0,b.length):参考以下说明。
3）read(byte[],int offset,int len)：最多读取len个字节，offset是数组中存储这些【字节】的起始位置。返回值是读取的【字节】数。这两个方法的内部还是调用read()方法来读取流，把读取的数组存储到传入的【字节数组】。

字符流

字符流是在字节流的基础上封装而来，允许用户可以在流中直接操作字符，而不必先计算一个字符对应几个字节，再从字节流中读取需要的组合字节。

PipedWriter

几个写入字符流的方法：
1）write(int a):a的低8位写入流中
2)write(char[] c):直接写入字符数组
3）write(String s):直接写入字符串
后两个方法最终同样是调用第1个方法完成流的写入(将字符变量强制转化为int)。

PipedReader

读取【字符】流：
1）read():返回一个流中的【字符】，如果没有数据返回-1
2）read(char[] c)->read(char[] c,0,c.length)
3)read(char[] c,int offset,int len):从【字符流】中最多读取len个字符到【字符数
offset是存储【字符】的起始位置。返回值是读取的【字符数量】，如果是-1表示没有【字符】可以读取了。
4）同样的，后2个方法也是调用第1个方法实现的。
